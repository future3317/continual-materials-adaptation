这轮是**真正有价值的突破**：D6 已经把 corrected PhyTCA 的平均 final nMAE 从 1.691 降到 0.548，并实现零遗忘。它明显优于当前 sequential FT 的 1.153，也比 D4/D5 的简单冻结父模型修正更好。D1–D6 至少证明了：

> **OPT 父路径固定 + MBJ 专属结构化残差**，比“OPT 和 MBJ 共同修改同一组 Tucker 因子”更适合这个材料多保真问题。

这也符合多保真材料学习的基本规律：低保真数据应作为高保真预测的父信息或共享表示，而不是被当成完全对称、彼此竞争的任务；已有多保真图网络研究也发现，加入低保真带隙可以显著改善高保真带隙预测。([arXiv][1])

但现在**还不能直接进入 5k×3 seeds**。我检查了论文和结果，发现当前诊断结论中还有四个必须先解决的逻辑问题。

# 一、当前最严重的问题：D3 并不能证明“顺序干扰”

论文表格中 D3 是：

[
T1@T1=2.409,\qquad T1@T2=2.409.
]

也就是说，D3 在只学完 Task 1 时就已经很差，Task 2 后旧任务没有进一步下降，报告的 forgetting 实际为零。

因此当前这句话：

> continual gap 远大于 architecture gap，所以失败来自 sequential interference

还不完全成立。

D2 与 D3 的差异可能来自：

* joint supervision 本身更容易优化；
* D3 的 Task-1 checkpoint 加载有问题；
* D3 只训练了错误的参数组；
* D3 的 OPT slice 在 Task 1 就被冻结或没有有效映射；
* joint 和 sequential 使用了不同 normalization、early stopping 或损失尺度；
* D3 的结果表误将 Task-2 后数值同时填入了 `T1@T1` 和 `T1@T2`。

要证明 sequential interference，必须看到：

[
T1@T1\ \text{正常},\qquad T1@T2\ \text{明显恶化}.
]

目前 D3 更像是：

> **sequential optimization failure on Task 1**

而不是已经严格定位到 Task-2 interference。

# 二、D4、D5、D6 并没有使用完全相同的 OPT 父模型

表中：

| 方法 | T1@T1 |
| -- | ----: |
| D4 | 0.817 |
| D5 | 0.791 |
| D6 | 0.706 |

如果三个实验都从**同一个冻结 OPT parent checkpoint**开始，而且 Task 2 不允许修改 OPT route，那么三者的 `T1@T1` 和 `T1@T2` 应该完全相同。

现在相差明显，说明至少有一个问题：

* Task-1 parent checkpoint 并不相同；
* D6 remapping 改变了有效 OPT 矩阵；
* 不同诊断方法重新训练了 Task 1；
* 不同模型结构导致 parent forward 并不等价；
* 统计来自不同初始化或不同 best epoch。

因此，在声称 D6 优于 D4/D5 之前，必须把比较变成：

```text
同一个 OPT checkpoint
同一个 OPT prediction hash
同一个 T1@T1
只改变 Task-2 correction module
```

否则 D6 的优势中可能包含一个更好的 Task-1 起点。

# 三、如果 OPT route 真正冻结，distillation 理论上应该不起作用

D6 被描述为：

> frozen OPT parent + MBJ residual + OPT distillation。

但如果：

* OPT encoder 路径冻结；
* OPT Tucker parent 冻结；
* OPT prediction head 冻结；
* Task 2 只训练 MBJ child residual；

那么：

[
\hat y_{\mathrm{OPT}}^{\mathrm{current}}(x)
===========================================

\hat y_{\mathrm{OPT}}^{\mathrm{teacher}}(x)
]

应始终成立，distillation loss 应接近机器精度下的零，(\lambda=0)、1、10 不应改变任何结果。

因此需要明确 D6 中到底还有什么共享参数在更新：

* graph encoder？
* shared output head？
* (U_{\rm in},U_{\rm out})？
* normalization affine parameters？
* property embedding？
* global core？
* readout？

如果某些共享参数仍会改变 OPT 输出，那么 D6 不是严格的 frozen-parent model，而是：

> **functionally consolidated parent + trainable shared representation**

这也可以成为一个好方法，但必须准确描述。

如果没有共享参数更新，而 (\lambda) 仍会改变结果，就说明 distillation 实现、teacher route 或优化器参数组仍有 bug。

# 四、“architecture gap 只有 0.126”表述过于乐观

D1 是 0.247，D2 是 0.373。虽然绝对差值是 0.126，但相对于 D1：

[
\frac{0.373-0.247}{0.247}\approx 51%.
]

这不是一个可以忽略的小差距。

它说明低秩 PhyTCA 容量是“有用的”，但仍明显弱于完整 joint model。尤其当前 encoder 是随机初始化，文献中参数高效适配通常依赖已经学到有效表征的预训练模型；材料领域的迁移学习研究也表明，在大规模形成能等数据上预训练晶体图网络，有助于后续带隙、模量和介电性质学习。([arXiv][2])

所以准确结论应该是：

> D2 证明低秩 adapter 具有基本容量，但仍存在明显 representation/architecture gap；D6 证明父子式 fidelity residual 能显著缩小 sequential gap。

# 五、D6 应当正式升级成新主方法

当前论文的方法部分仍主要描述：

* 对称的 property/fidelity Tucker slice；
* group-lasso；
* shared-factor stability loss；
* 冻结旧 slice；
* 梯度残差扩 rank。

但真正有效的 D6 已经不是这个方法。

建议将主方法重构为：

# FR-PhyTCA

## Fidelity-Residual Physics-Structured Tensor Component Adaptation

## 1. 父保真度模型

先学习低成本、数据丰富的 OPT：

[
h_{\mathrm{OPT}}(x)
===================

f_{\theta}(x),
]

[
\hat y_{\mathrm{OPT}}
=====================

g_{\mathrm{OPT}}
\left(
h_{\mathrm{OPT}}(x)
\right).
]

Task 1 后保存并冻结有效父路径：

[
\Theta_{\mathrm{OPT}}^\star.
]

## 2. 高保真残差模型

MBJ 不再作为平行 fidelity slice，而是建模为：

[
\hat y_{\mathrm{MBJ}}
=====================

\hat y_{\mathrm{OPT}}
+
\delta_{\phi}(x).
]

其中残差由 progressive Tucker adapter 生成：

[
\delta_{\phi}^{(\ell)}(H)
=========================

U_{\mathrm{out}}^{\Delta}
R_{\mathrm{MBJ}\leftarrow\mathrm{OPT}}^{(\ell)}
\left(U_{\mathrm{in}}^{\Delta}\right)^\top H.
]

MBJ 模型的有效更新是：

[
A_{\mathrm{MBJ}}
================

A_{\mathrm{OPT}}^\star
+
A_{\mathrm{MBJ}\leftarrow\mathrm{OPT}}^\Delta.
]

## 3. 严格父路径不变性

定义父路径相关参数集合：

[
\Theta_P.
]

在学习 MBJ 时要求：

[
\Theta_P^{(2)}
==============

\Theta_P^{(1)}.
]

因此得到一个比旧 stability bound 更强的结论：

### Parent-route invariance

对于任意输入晶体 (x)：

[
f_{\mathrm{OPT}}
\left(x;\Theta^{(2)}\right)
===========================

f_{\mathrm{OPT}}
\left(x;\Theta^{(1)}\right).
]

于是：

[
F_{\mathrm{OPT}}=0.
]

这不是正则化意义上的“有界遗忘”，而是结构保证的**精确零遗忘**。

## 4. 可选共享更新

只有在纯冻结 residual 的 MBJ 性能不足时，才开放一个正交的新子空间：

[
U_\Delta^\top U_{\mathrm{OPT}}=0.
]

这与 O-LoRA、InfLoRA 的思想一致：新任务增量应落入与旧任务低干扰的低秩子空间，而不是继续旋转旧任务的有效参数。([开放评审][3])

# 六、旧理论应该重写，而不是继续保留

当前 Theorem 3.3 在正文和证明中仍不一致：

* 定理正文使用新任务梯度在 (\Theta^{(t-1)}) 处；
* 附录推导使用梯度在 (\Theta^{(t)}) 处。

而且实验证明 L2 stability 不是有效主机制，因此不应该让它继续成为论文理论中心。

新理论建议只保留三个清晰命题：

### Proposition 1：父路径精确不变性

冻结父路径后：

[
\forall x,\quad
\hat y_{\mathrm{OPT}}^{\mathrm{after}}(x)
=========================================

\hat y_{\mathrm{OPT}}^{\mathrm{before}}(x).
]

### Proposition 2：增量参数增长

若每个新 fidelity 只增加 rank-(r_f) residual：

[
\Delta P_f
==========

r_f(d_{\mathrm{in}}+d_{\mathrm{out}})
+
r_f^2,
]

总模型增长与 residual rank 成正比，而不是复制完整模型。

### Proposition 3：高保真误差分解

[
\left|
y_{\mathrm{MBJ}}-\hat y_{\mathrm{MBJ}}
\right|
\leq
\left|
y_{\mathrm{OPT}}-\hat y_{\mathrm{OPT}}
\right|
+
\left|
\Delta y-\widehat{\Delta y}
\right|.
]

这直接说明：

* 好的 OPT parent 很重要；
* MBJ 学习的难度取决于 fidelity correction；
* random backbone 会限制 residual 模型上限。
