# 总体判断

我通读了论文和技术说明，并检索了截至 **2026 年 7 月 15 日**公开的张量分解、持续学习、多保真学习、材料图网络和晶体张量预测研究。

**这个方向是合理的，而且实验中确实存在值得继续追的信号；但当前版本还不适合直接按现有标题和贡献表述投 ICLR 2027 主会。**

目前最有价值的结果不是“用了 Tucker”或“证明了零遗忘”，而是：

> **在严格保持低保真模型输出不变的约束下，用约 4k 新增参数显著改善高保真预测。**

你们从对称共享 Tucker 失败，到诊断顺序优化干扰，再改成冻结父路径的 residual child，这个研究过程是扎实的。5k×3 seeds 中 FR-PhyTCA 的 MBJ nMAE 为 0.342，而参数量相近的 MLP 和低秩 residual 约为 0.829–0.831，信号很强；父路径漂移为零也被验证了。 

不过，现在有四个会被审稿人迅速抓住的核心问题：

1. 一个理论解释是错误的；
2. 参数增长命题与实际实现、实验参数数目不一致；
3. 当前单性质、单新增 fidelity 实验中，Tucker 实际退化成普通低秩映射；
4. “physics-structured”“task-ID free”“continual database evolution”等表述超过了当前实现和实验所证明的范围。

我的判断是：**idea 可以救，而且有机会升级成更强的工作，但需要把论文中心从“冻结产生零遗忘”转向“怎样在不可修改旧模型的条件下，高效学习具有物理层级关系的新 fidelity”。**

# 一、目前做得比较好的部分

你们最好的地方是没有隐藏原始 PhyTCA 的失败，而是明确报告 Phase 0 NO-GO，并通过 D1–D6 分解容量限制、随机 backbone、顺序优化失败和父子 fidelity 关系。这比直接展示一个最终表格更有研究价值。

特别是以下几点是可以保留的：

* 所有 residual 方法共享同一 OPT parent checkpoint，减少了初始化造成的比较偏差；
* 父路径预测 hash、state-dict hash 和输出漂移被显式检查；
* final test 没有用于调参；
* Protocol B 对 paired fidelity 使用相同 split；
* 对周期图和数据协议做了单元测试与 GO/NO-GO 审计；
* FR-PhyTCA 在 2k 和 5k、三个随机种子上保持优势。

但是这些结果目前只证明：

> 在你们当前 EGNN 表征、当前 OPT→MBJ band-gap 设置、当前 adapter placement 下，某个 Tucker-style residual 比所实现的两个 residual 基线好。

它还没有证明：

* 高阶 Tucker 维度带来了多轴共享；
* 方法适用于长任务序列；
* 方法解决了真正的数据库版本演化；
* 方法中的“physics structure”产生了收益；
* 方法优于架构完全匹配的 LoRA；
* 方法在强材料 backbone 上仍然有效。

# 二、必须首先修正的数学问题

## 2.1 Fidelity-error decomposition 的结论与当前训练目标不一致

你们当前模型和损失是：

[
\hat y_H(x)=\hat y_L(x)+\delta_\phi(x),
]

[
\mathcal L_H=
\left(y_H-\hat y_L-\delta_\phi\right)^2.
]

因此 residual 实际学习的目标是：

[
r^*(x)=y_H(x)-\hat y_L(x),
]

而不是物理意义上的：

[
\Delta y(x)=y_H(x)-y_L(x).
]

于是高保真误差有一个更直接的恒等式：

[
y_H-\hat y_H
============

## \underbrace{(y_H-\hat y_L)}_{r^*}

\delta_\phi.
]

也就是说：

[
|y_H-\hat y_H|=|r^*-\delta_\phi|.
]

如果 residual 完美学到了 (r^*=y_H-\hat y_L)，即使 parent 本身有误差，高保真误差仍然可以是零。论文当前“即使 residual 完美，高保真误差也不能低于 parent error”的解释，在现有损失下不成立。论文中的三角不等式只是一个**上界**，也不能推出 parent error 是下界。

### 推荐的理论改法

把当前方法明确称为 **prediction-residual learning**，定义：

[
q_H(x)=y_H(x)-\hat y_L(x).
]

然后给出风险恒等式：

[
\mathcal R_H(\phi)
==================

\mathbb E
\left[
(y_H-\hat y_L-\delta_\phi)^2
\right]
=======

\mathbb E
\left[
(q_H-\delta_\phi)^2
\right].
]

真正值得分析的是：

* parent 越好，(q_H) 是否具有更小方差；
* correction 是否具有更低的有效秩；
* correction 在冻结表征上是否更平滑；
* rank-(r) adapter 对 correction operator 的近似误差是多少。

例如，在冻结表示 (h(x)) 上假设最优 correction 为线性映射 (W^*h(x))，rank-(r) adapter 的最优近似误差可以和 (W^*) 的尾部奇异值联系起来：

[
\min_{\operatorname{rank}(W)\le r}
|W^*-W|_F^2
===========

\sum_{i>r}\sigma_i^2(W^*).
]

这比当前的三角不等式更能解释：

* 为什么某些 fidelity 转换适合低秩 residual；
* 为什么某些任务需要扩大 rank；
* 怎样做自适应 rank；
* 为什么 parent 与 child 的物理接近程度会影响样本效率。

另一条路线是改成真正的物理 (\Delta)-learning：

[
\delta_\phi \approx y_H-y_L.
]

但这样必须用配对的真实 (y_L,y_H) 监督 residual，并且要把它与当前预测校正式 residual 分开做实验。多保真材料建模中，(\Delta)-learning 已经是成熟基线，不能把它当作隐含的新贡献。([arXiv][1])

## 2.2 参数增长命题与实际 3,923 个参数不一致

论文命题声称：

[
\Delta P_{\text{fid}}
\le
L(2dr+r^2)+d.
]

取实验配置：

[
L=3,\quad d=64,\quad r=8,
]

得到：

[
3(2\cdot64\cdot8+8^2)+64=3328.
]

即使加 head bias，也只有 3329。

但论文实验报告的新增参数是 **3923**。根据技术文档中的实际实现，每层 child Tucker 包含：

* (U_{\text{out}})：(dr)
* (U_{\text{in}})：(dr)
* core：(r^2R_pR_f)
* property embedding：(N_pR_p)
* fidelity embedding：(N_fR_f)

所以实际参数公式应是：

[
\Delta P
========

L\left(
2dr+r^2R_pR_f+N_pR_p+N_fR_f
\right)
+(d+1).
]

代入当前配置：

[
R_p=R_f=2,\quad N_p=1,\quad N_f=2,
]

得到：

[
3(1024+256+2+4)+65
==================

3923.

]

这正好等于表中的数目。也就是说，当前命题所写的“上界”小于实际参数数目，因此命题在现有实现下是错误的。 

有两种修复方式：

1. **修正文中的参数公式**，如实包含 core、embedding 和 bias；
2. **简化实现以符合原命题**：对于一个新 fidelity，直接学习每层的
   [
   U_{\text{out}}M_fU_{\text{in}}^\top,
   ]
   不再为单个 child 创建完整 (R_p\times R_f) Tucker core 和 embedding。

第二种实现不仅更干净，也能引出下一个更重要的问题。

## 2.3 当前实验中 Tucker 实际退化了

对于固定的一个 property 和一个 fidelity：

[
M_{p,f}
=======

\mathcal G\times_3 e_p\times_4e_f
\in\mathbb R^{r_{\text{out}}\times r_{\text{in}}},
]

所以 adapter 实际是：

[
\Delta W_{p,f}
==============

U_{\text{out}}M_{p,f}U_{\text{in}}^\top.
]

在目前主要实验只有：

* 一个 property：band gap；
* 一个 parent fidelity：OPT；
* 一个新增 fidelity：MBJ；

因此 property 和 fidelity 两个 Tucker 模式没有进行可验证的跨任务共享。它在函数形式上只是一个带可训练中间矩阵的低秩 adapter。

这意味着现在的实验无法支持：

> “高阶 Tucker 结构优于普通低秩结构。”

尤其是你们报告 FR-PhyTCA 的 T2 nMAE 为 0.342，而“matched low-rank residual”为 0.831，差异过大。第一反应不会是 Tucker 有巨大优势，而会是：

* 两者是否插在同样的层；
* 是否都作用于每个 message-passing block；
* 是否都使用相同 hidden features；
* 是否都有可训练中间矩阵；
* rank、初始化、归一化、优化器参数组是否一致；
* low-rank baseline 是否实际上只放在 readout 或单层。

必须增加以下架构完全一致的基线：

[
\text{LoRA-AB}: \quad U_{\text{out}}U_{\text{in}}^\top,
]

[
\text{LoRA-ABA}: \quad U_{\text{out}}M U_{\text{in}}^\top,
]

[
\text{FR-Tucker}: \quad
U_{\text{out}}
(\mathcal G\times e_p\times e_f)
U_{\text{in}}^\top.
]

三者必须具有：

* 相同插入位置；
* 相同 parent hidden representation；
* 相同可训练参数预算；
* 相同训练步数和 batch 顺序；
* 相同 parent checkpoint；
* 相同初始化尺度。

只有在至少两个 property、三个以上 fidelity 或多个 fidelity transition 上，Tucker 的多轴共享才真正可识别。

这点尤其重要，因为低张量秩 PEFT 已有 LoTR，而 2026 年的 TuKA 已经直接使用 Tucker 表示多层级知识并进行 lifelong adaptation。因而“将 Tucker 用于多轴持续适配”本身已经很难作为主要新颖性。([arXiv][2])

## 2.4 Parent-route invariance 正确，但理论贡献较弱

冻结 parent 参数、让 optimizer 更新集合与 parent 不相交，当然可以得到：

[
f_L(x;\Theta^{(t+1)})=f_L(x;\Theta^{(t)}).
]

这个结论是正确的，但它本质上是结构隔离的直接推论。Progressive Neural Networks 等模块化方法很早就通过冻结旧列实现了无遗忘；近年的 O-LoRA、SD-LoRA、PEARL、RA-LoRA 等则研究正交子空间、方向/幅值解耦和动态 rank。([arXiv][3])

所以不要把“零遗忘定理”作为最大理论卖点。更强的理论应回答：

* 在固定参数预算下，哪一个 parent 最适合作为新 fidelity 的父节点；
* fidelity residual 的有效秩如何影响误差；
* rank 应怎样随任务复杂度增长；
* 多条 fidelity 路径之间何时一致；
* 参数增长、预测误差和计算成本之间有什么 Pareto bound；
* 不配对数据下还能否识别 fidelity correction。

另外，严格不变性需要明确涵盖的不只是 `requires_grad=False`：

* optimizer state 不包含 parent 参数；
* weight decay 不触碰 parent；
* BatchNorm running statistics 不更新；
* dropout/随机 routing 不改变 parent 输出；
* preprocessing 和目标归一化不改变；
* parent forward 处于确定性模式；
* child 不共享任何会被修改的 buffer。

# 三、结合最新文献后，当前新颖性应该怎样定位

## 3.1 计算机科学与张量方法

近年的方向已经从“每任务一个固定 rank LoRA”发展到：

* 正交或几何约束的低秩子空间；
* rank 随任务动态增长；
* 方向与幅值解耦；
* 多层或多轴共享张量；
* 对旧任务子空间进行显式隔离。

TuKA 已经利用 Tucker 分解表示 scene、environment 等多层级持续知识；LoTR 把参数更新表示成低张量秩；2026 年的 PEARL、RA-LoRA 等进一步做 task/layer-specific rank allocation。([arXiv][4])

因此你们不能只把贡献写成：

> Tucker decomposition + continual learning + zero forgetting。

需要突出材料 fidelity 之间的科学结构、样本成本和层级依赖。

## 3.2 数学上的 fidelity graph 也已有先例

将多个 fidelity 表示为 DAG 并不是空白。Graphical Multi-Fidelity Gaussian Process 已经把科学模型之间的依赖表示为有向无环图。([arXiv][5])

所以升级成 fidelity DAG 时，真正的新颖点应是组合：

* fidelity 随时间持续到达；
* 旧节点必须精确不变；
* 节点使用神经原子表示；
* 自动选择父节点；
* 自适应分配 residual rank；
* 支持不配对数据；
* 多路径 consistency；
* 成本感知的数据采集。

## 3.3 多保真材料学习和 (\Delta)-learning 已较成熟

2020 年的 multi-fidelity graph network 已经展示低 fidelity band gap 数据对高 fidelity 或实验 band gap 的帮助；多保真 correction、co-kriging 和 (\Delta)-learning 在 band-gap 和原子势研究中也已有较长历史。([arXiv][6])

一项 2025 年的多保真机器学习力场研究还发现：

* 低 fidelity 预训练可以显著帮助低数据高 fidelity 学习；
* backbone 学到的表示往往与计算方法相关；
* 仅冻结 backbone 通常不如完整 fine-tuning；
* 多头训练可学习更通用表示，但可能牺牲目标 fidelity 的最优性能。([arXiv][1])

这与你们的实验非常吻合：随机且冻结的 encoder 会成为高 fidelity 的明显瓶颈。它也说明只在随机 frozen EGNN 上证明 residual 有效，不足以支撑材料建模层面的强结论。

## 3.4 “Physics-Structured”目前没有落地

论文提到点群投影和张量性质，但当前代码只实现 formation energy 和 band gap 标量任务，具体的 tensor symmetry projector 尚未实现。

与此同时，晶体张量预测已经进入直接 E(3)/O(3)-equivariant 建模阶段，覆盖 dielectric、piezoelectric 和 elastic tensors；2026 年工作还在进一步提高高阶张量预测效率。([arXiv][7])

因此单纯在输出后做 point-group projection 只能作为最低限度的合法性约束，不足以成为“physics-structured”的主要依据。至少应实现以下一种：

* irreducible representation / equivariant tensor head；
* 点群作用下的群平均投影；
* Voigt notation 与 minor/major symmetry 的严格约束；
* centrosymmetry 下 piezoelectric tensor 严格为零；
* 对称性约束前后精度和物理违规率对比。

在这些功能完成前，更稳妥的是暂时从标题中移除 `Physics-Structured`。

# 四、我建议的方法升级路线

可以升级成：

> **Continual Fidelity Graph Adaptation with Exact Retention**

设 fidelity 构成图 (G=(V,E))。每个新节点 (v) 从一个或多个旧 fidelity 父节点预测：

[
\hat y_v(x)
===========

\sum_{u\in \mathrm{Pa}(v)}
\alpha_{uv}(x)
\left[
\hat y_u(x)+r_{uv}(h_u(x))
\right].
]

其中：

* 所有旧节点和旧 edge residual 永久冻结；
* 新 fidelity 只增加新的 edge modules；
* (\alpha_{uv}) 可以由成本、验证 residual、结构域或不确定性决定；
* 每条 edge 有独立或共享的低秩子空间；
* rank 根据 residual spectrum 或梯度 principal angle 自动增长；
* 多条路径通过 path-consistency loss 对齐。

需要重点加入四个组件。

### 1. 自动父节点选择

不是默认永远 OPT→MBJ，而是在候选 parent 中评估：

[
S(u\rightarrow v)
=================

\text{validation residual error}
+
\lambda_P P_{uv}
+
\lambda_C C_u,
]

其中 (P_{uv}) 是增量参数或 residual complexity，(C_u) 是获取该 fidelity 的计算成本。

### 2. 自适应 rank

根据 residual operator 的谱尾部选择最小 rank：

[
r^*
===

\min
\left{
r:
\frac{\sum_{i>r}\sigma_i^2}
{\sum_i\sigma_i^2}
\le\epsilon
\right}.
]

或者用新任务梯度子空间与已有子空间的 principal angle 决定复用还是扩展。近期低秩持续学习也正在转向动态 rank 和子空间几何，因此必须用材料 fidelity 结构来区别你们的方法。([arXiv][8])

### 3. Copy-on-write 高层模块

你们的 D6-f 已经给出了很重要的线索：更新 top encoder layer 改善了新任务，但导致旧 route 漂移。

不要更新共享 top layer；改为：

* 从 parent top block 复制一个 child-private block；
* parent 继续使用原 block；
* high fidelity 使用复制后的 block；
* 只更新 child copy；
* 再对 child copy 做低秩化、剪枝或蒸馏。

这样理论上仍然 exact retention，同时允许方法特定的高层表示适配。建议比较：

1. residual adapter only；
2. private top block；
3. private top block + low-rank compression；
4. private last (k) blocks；
5. adaptive selection of copied blocks。

这很可能比单纯继续增加 Tucker rank更有效，因为近期多保真研究也显示 backbone 表示具有 fidelity-specific 特征。([arXiv][1])

### 4. 多路径一致性

例如同时存在：

[
\mathrm{PBE}\rightarrow\mathrm{MBJ},
]

以及：

[
\mathrm{PBE}\rightarrow\mathrm{SCAN}\rightarrow\mathrm{MBJ}.
]

可以加入：

[
\mathcal L_{\text{path}}
========================

\left|
\hat y_{\mathrm{PBE}\rightarrow\mathrm{MBJ}}
--------------------------------------------

\hat y_{\mathrm{PBE}\rightarrow\mathrm{SCAN}\rightarrow\mathrm{MBJ}}
\right|^2.
]

这个组件能使 fidelity graph 不只是多个独立 adapter 的集合，而有真正的图结构学习。

# 五、实验必须怎样重做

## P0：提交前必须完成的正确性审计

### 5.1 核对跨 fidelity 归一化

技术文档说明每个任务独立计算 mean/std。如果代码中直接执行：

[
\hat y_H^{\rm norm}
===================

\hat y_L^{\rm norm}+\delta^{\rm norm},
]

那么两个量可能处在不同的归一化坐标系中，这是严重错误。

正确做法是首先恢复物理单位：

[
\hat y_L^{\mathrm{eV}}
======================

\sigma_L\hat y_L^{\mathrm{norm}}+\mu_L,
]

然后定义 residual：

[
r_H^{\mathrm{eV}}
=================

y_H^{\mathrm{eV}}-\hat y_L^{\mathrm{eV}},
]

最后：

[
\hat y_H^{\mathrm{eV}}
======================

\hat y_L^{\mathrm{eV}}+\hat r_H^{\mathrm{eV}}.
]

可以单独对 (r_H^{\mathrm{eV}}) 做标准化，但 parent-child 的加法必须在同一单位中完成。

由于目前没有实际 `diagnostics.py`，还不能确认代码是否真的存在这个错误；但这是源码逐行审计的第一优先级。

### 5.2 全局防止跨版本泄漏

当前 Protocol B 保证同一年 OPT/MBJ paired record 共享 split，但还需要保证：

* 2021 与 2022 中相同 JID 始终属于同一 split；
* 相同结构不同 JID 始终属于同一 split；
* primitive/conventional/supercell 表示的同一材料不跨 split；
* 轻微弛豫版本和近重复结构不跨 split；
* label revision 不会导致旧版本 train 和新版本 test 实际包含同一材料。

建议先建立全局 canonical material group：

[
\text{group_id}
===============

\text{composition}
+
\text{StructureMatcher/structure fingerprint cluster},
]

再一次性给 group 分配 train/val/test，所有年份、property 和 fidelity 继承该 split。

Formula-disjoint 不能排除同化学式不同多晶型，也不能排除接近重复的结构。

### 5.3 Protocol A 应包含真正的“版本修订”

当前 A2/A4 只使用 2022 中新增 JID，因此没有覆盖：

* 同一 JID 的 label 被修正；
* 同一结构重新弛豫；
* 元数据或计算设置改变；
* 旧估计被高精度结果替换。

建议把数据库 diff 分为：

1. new material；
2. revised structure；
3. revised target；
4. newly available property；
5. changed fidelity；
6. deleted/deprecated record。

否则 Protocol A 更像“新增样本继续训练”，而不是完整的数据库演化。

### 5.4 核对 Raw MAE 列

Table 4 中：

* D6 的 T2 nMAE 为 0.446，Raw MAE 为 0.996；
* D6-f 的 T2 nMAE 为 0.342，Raw MAE 为 1.142。

如果 Raw MAE 是同一个 T2 的 MAE，固定 MAD 下更低 nMAE 不可能对应更高 MAE。更可能的情况是 Raw MAE 是两个任务平均值，但表头没有写清楚。

建议分别报告：

* OPT MAE；
* MBJ MAE；
* OPT nMAE；
* MBJ nMAE；
* macro-average；
* 每个任务的 MAD denominator。

不要把不同任务的 eV MAE 平均后称为单一的“Raw MAE”，除非明确说明。

### 5.5 架构完全匹配的 adapter baselines

至少需要：

* per-layer LoRA；
* LoRA with trainable middle matrix；
* bottleneck adapter；
* residual MLP；
* independent Tucker；

* copy-on-write top block；
* full child backbone；
* joint multi-head；
* classical (\Delta)-learning；
* multi-fidelity graph network；
* co-kriging/GP，至少在冻结结构 embedding 上实现。

所有 PEFT 方法要统一：

* backbone；
* adapter location；
* hidden dimensions；
* parent checkpoint；
* parameter count；
* optimizer、scheduler；
* early stopping；
* batch sequence；
* hyperparameter search budget。

### 5.6 使用强 backbone

随机初始化且冻结的 EGNN 是当前工作最明显的科学有效性瓶颈。JMP 在广泛原子/材料任务上显示了预训练相对于从头训练的明显收益；MatGL 也已经提供 invariant、equivariant 模型、预训练势和 property models。([arXiv][9])

建议至少测试三个层级：

1. 当前小 EGNN，便于机制分析；
2. ALIGNN/MatGL property backbone；
3. JMP 或其他预训练原子 backbone。

然后比较：

* frozen backbone；
* adapter-only；
* private top block；
* full child fine-tuning；
* joint training upper bound。

如果 FR-PhyTCA 只在随机冻结 backbone 上有优势，而在强 backbone 上被 LoRA 或简单 fine-tuning 消除，就不应继续强调 Tucker。

### 5.7 使用完整数据规模

5k 并不能称为“realistic fidelity scaling”，因为 Protocol B 中已有约 18k/20k paired records。主结果至少应使用完整训练集。

建议样本效率曲线采用真实不平衡设置：

[
N_L=10k,30k,50k,\qquad
N_H=100,500,1k,2k,5k,\text{all}.
]

核心图应该是：

* high-fidelity MAE vs. high-fidelity labels；
* MAE vs. incremental parameters；
* MAE vs. training FLOPs；
* MAE vs. stored checkpoint bytes。

真实场景下低 fidelity 应明显多于高 fidelity，而不是 2k→2k、5k→5k。

### 5.8 长序列与真正的多轴测试

当前主实验只有两步 OPT→MBJ。至少应形成：

[
f_1\rightarrow f_2\rightarrow f_3\rightarrow f_4,
]

并报告完整 accuracy matrix。

理想设置是：

* fidelity：PBE/OPT、SCAN、MBJ、HSE 或 experiment；
* property：formation energy、band gap、bulk modulus、dielectric；
* database/domain：JARVIS、Materials Project 或其他来源。

要证明 Tucker 多轴结构，至少要有：

[
N_p\ge 2,\qquad N_f\ge 3,
]

并展示：

* shared factor 是否被多个 property/fidelity 使用；
* 移除 property mode 后性能是否下降；
* 移除 fidelity mode 后性能是否下降；
* 打乱 property/fidelity 标签是否破坏收益；
* component reuse 和物理相关性是否一致。

### 5.9 Paired、partially paired 和 unpaired 三种设置

当前 residual 依赖 paired OPT/MBJ 结构。现实中不同 fidelity 数据通常不完全重合。

建议分别设置：

* 100% paired；
* 50% paired；
* 10% paired；
* 完全不配对。

在 unpaired 情形，可以比较：

* multi-head shared representation；
* distribution alignment；
* pseudo-pairing；
* teacher prediction correction；
* graph-based fidelity transfer。

近期多保真工作表明，多头模型即使不同 fidelity 的结构集合不重合，也可能有效，因此这是重要基线。([arXiv][1])

# 六、代码层面的具体优化

目前只有技术说明，没有实际 `data.py`、`phytca.py`、`diagnostics.py` 和 `baselines.py`，所以下面是架构级审核，不是逐行结论。

## 6.1 不要显式构造 (d\times d) 的 `delta_w`

当前实现：

```python
core_slice = torch.einsum("oipf,p,f->oi", G, e_p, e_f)
delta_w = U_out @ core_slice @ U_in.t()
out = x @ delta_w.t()
```

可以利用结合律：

```python
import torch.nn.functional as F

def apply_factorized_update(x, u_in, core_slice, u_out):
    # u_in:       [d_in, r_in]
    # core_slice: [r_out, r_in]
    # u_out:      [d_out, r_out]

    z = F.linear(x, u_in.T)       # x @ U_in
    z = F.linear(z, core_slice)   # z @ core_slice.T
    return F.linear(z, u_out)     # z @ U_out.T
```

这样避免物化 `delta_w`。

原实现主要计算量近似为：

[
O(d^2r+Nd^2),
]

结合式实现近似为：

[
O(Ndr+Nr^2).
]

在 (d=64) 时差异可能不大，但换强 backbone 或更大 hidden dimension 后会明显。

如果一个 batch 中所有样本属于同一 property/fidelity，`core_slice` 每个 batch 只计算一次。混合任务 batch 则可以构造：

[
M\in\mathbb R^{B\times r_{\text{out}}\times r_{\text{in}}},
]

然后使用 `torch.bmm`，而不是 Python 循环。

## 6.2 单 child 时不要保留冗余 Tucker modes

对于只有一个新增 fidelity 的 child，直接定义：

```python
self.u_in = nn.Parameter(torch.empty(d_in, rank))
self.middle = nn.Parameter(torch.empty(rank, rank))
self.u_out = nn.Parameter(torch.empty(d_out, rank))
```

即可。

只有在确实有多个 property/fidelity 共享同一个 child bank 时，再启用：

* `core`
* `property_embedding`
* `fidelity_embedding`

这样可以同时：

* 修复参数命题；
* 减少参数；
* 清楚地区分普通低秩 residual 与多轴 Tucker residual；
* 使实验更容易解释。

## 6.3 用结构隔离代替 gradient zeroing

不要创建一个包含 parent slice 的 child core，再靠：

```python
grad[parent_slice] = 0
```

来保持不变。

更稳妥的设计是：

```python
self.parent = FrozenParent(...)
self.children = nn.ModuleDict({
    "mbj": ResidualAdapter(...),
    "hse": ResidualAdapter(...),
})
```

parent fidelity 的 forward 根本不经过 `children`。新 fidelity optimizer 只接收：

```python
optimizer = AdamW(
    model.children["mbj"].parameters(),
    ...
)
```

这样：

* Adam momentum 不可能修改 parent；
* weight decay 不可能修改 parent；
* 不需要 gradient hook；
* 不需要每 step 重置切片；
* exact-retention theorem 与实现完全一致。

child 训练时 parent forward 可在无梯度模式下运行：

```python
model.parent.eval()
with torch.no_grad():
    parent_pred, parent_features = model.parent(batch)
```

要注意，如果 child 需要中间层特征，应明确决定是否缓存这些特征；不要让 dropout 或可变 buffer 破坏输出确定性。

## 6.4 周期图需要重构

`2×2×2` supercell + dense kNN EGNN 存在三个问题：

1. 超胞显著增加节点数和二次复杂度；
2. 可能重复选择同一原子的多个镜像；
3. `update_coors=True` 会让相同原子的不同周期镜像独立移动，从而破坏周期一致性。

建议改成显式周期边：

* 节点只保存原胞原子；
* 边记录 `(src, dst, lattice_shift)`；
* 相对位移计算为
  [
  r_{ij}=x_j+L n_{ij}-x_i;
  ]
* 根据 cutoff 而不是固定 kNN 构边；
* pooling 直接在原胞节点上完成。

对于 formation energy 和 band gap 这类静态结构标量预测，优先将 `update_coors=False`。除非模型显式保证周期坐标更新一致，否则没有必要让 EGNN改变输入晶体坐标。

MatGL 已提供基于 pymatgen/DGL 的材料图构建、invariant/equivariant 模型和训练流程，适合替换当前自定义超胞 padding 管道。([Nature][10])

## 6.5 使用稀疏图 batch，不要 padding 成密集点云

建议迁移到：

* PyG `Data/Batch`；
* DGL graph batch；
* 或 MatGL dataset。

每个图只保存真实边和节点，避免：

* padding mask；
* 密集 (N\times N) 邻接计算；
* 超胞造成的大量虚拟节点；
* batch 内按最大原子数扩展。

预处理后的 periodic graph 可以缓存到：

* LMDB；
* PyG `OnDiskDataset`；
* memory-mapped shards。

训练时配合：

* `pin_memory=True`
* `persistent_workers=True`
* 合理的 `prefetch_factor`
* BF16 autocast
* `torch.compile`
* gradient accumulation

但应先 profile graph construction、collation、forward 和 adapter contraction，不能盲目优化。

## 6.6 TensorLy 不应直接替换当前核心 forward

TensorLy-Torch 确实提供 factorized tensor layers、Tucker layers 和 contraction layers。([TensorLy][11])

但你们的场景需要：

* fidelity-specific routing；
* frozen parent；
* progressive child；
* 动态新增任务；
* exact parameter accounting；
* 特殊的 rank allocation。

因此不建议用 `tucker_to_tensor` 重构完整权重张量；那会抵消低秩计算优势。更适合的用途是：

* 验证手写 contraction 的数值正确性；
* Tucker/SVD 初始化；
* rank regularization；
* 与标准 tensorized layer 做对照。

核心 adapter 保留简洁的自定义 PyTorch 实现更好。

## 6.7 Hugging Face PEFT 不一定适合这个项目

`peft` 主要面向 Transformer 风格模块自动注入。你们需要在 GNN message-passing block 的特定位置插入 residual，并管理 fidelity graph，手写小型 LoRA/Tucker module 反而更清晰。

更重要的是建立统一接口，例如：

```python
class ResidualAdapter(nn.Module):
    def forward(self, x, context): ...
    def incremental_parameter_count(self) -> int: ...
    def route_signature(self) -> str: ...
```

让 LoRA、MLP、Tucker、copy-on-write block 都通过同一训练和评估管道运行。

# 最终结论

**Idea 的物理动机和工程动机是成立的，FR redesign 也明显优于最初的对称 PhyTCA。**但当前实验最强的结论只能是：

> 冻结 OPT parent，并在当前 EGNN 的多层 hidden representation 上加入一个约 4k 参数的 Tucker-style correction，可以在不改变 OPT 输出的情况下，提高 MBJ band-gap 预测。

目前还不能得出：

> Tucker 多轴分解是优势来源；
> 该方法解决了一般的多轴持续学习；
> 该方法具有真正的 physics structure；
> 该方法能够普遍处理持续演化的材料数据库。

最值得升级的方向是：

> **fidelity graph + exact retention + adaptive rank + child-private representation adaptation + realistic high/low fidelity imbalance。**

这条路线既利用了你们已经观察到的 FR 信号，也避开了“冻结自然导致零遗忘”和“单任务 Tucker 退化”两个新颖性陷阱。
